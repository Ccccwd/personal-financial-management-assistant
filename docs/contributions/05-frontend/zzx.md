## 遇到的问题和解决

1. **问题**：TypeScript 强类型校验失败 `类型“AxiosResponse”上不存在属性“budgets”`。
   **解决**：由于项目底层的 Axios 请求拦截器已统一对 `res.data` 进行了剥离化处理，返回的直接是实际业务数据，绕过了原来的接口定义层谱系。在进行响应接收赋值时，加入 `(res as any)` 使其适配现有的类型上下文。

2. **问题**：查询表单使用 Element Plus 日历选择器导致传参类型报错 `不能将类型“string”分配给类型“number”`。
   **解决**：在触发查询 API 传递参数的阶段，进行手动的强制类型转换以符合接口规约（`year: Number(queryForm.year)`）。

3. **问题**：在访问“账户管理”和“预算管理”等核心金额计算页时，页面陷入一直加载的状态（白屏死机），Console 控制台深层抛出 `Uncaught (in promise) TypeError: ...toFixed is not a function`。
   **解决**：深度排查发现，后端为了财务数据的绝对精度，将相关数值类型序列化以字符串 `String` 下发，而 Vue 模板原本默认它为 JS 数字对象并直接调用了 `.toFixed()` 方法。使用正则表达式批量定位了全局存在的类似代码，统一改写替换为 `Number(value || 0).toFixed(2)` 的包裹形式。
